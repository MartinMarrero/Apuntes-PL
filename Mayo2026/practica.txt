program -> decls ';' expr | expr

decls   -> decl ';' decls | decl

decl    -> type id

expr    -> expr ';' expr

        | expr '-' expr
        | expr '*' expr
        | expr '/' expr
        | id '=' expr
        | '(' expr ')'
        | '{' decls ';' expr '}'
        | '{' expr '}'
        | num
        | id

2. a. Is the grammar ambiguous? If so, propose an unambiguous one.

Sí, la gramática es ambigua, pues no establece un orden de precedencias para prácticamente ninguna operación. Por ejemplo, la siguiente línea:

a + b * c

El compilador podría interpretarla como:

(a + b) * c

o

a + (b * c)

La propuesta para una gramática no ambigua es:

program     -> decls ';' add_expr 
            | add_expr

decls       -> decl ';' decls 
            | decl

decl        -> type id

type        -> int 
            | float

assign_expr -> id '=' assign_expr
            | add_expr

add_expr    -> add_expr '+' mult_expr
            | add_expr '-' mult_expr
            | mult_expr

mult_expr   -> mult_expr '*' unary_expr
            | mult_expr '/' unary_expr
            | unary_expr

unary_expr  -> '-' primary
            | primary

primary     -> '(' assign_expr ')'
            | '{' decls ';' assign_expr '}'
            | '{' assign_expr '}'
            | num
            | id
